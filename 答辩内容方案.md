## 1. 叙事主线（一句话定位）

> **「星秩存册，是学习的图书馆，也是"健身房"。」**
> 它用一套"可信、可累积、跨机构流通"的学分银行，把零散的学习变成有氛围、有激励、有认证的成长。

三幕逻辑：
1. **为什么做** —— 终身学习有"脱节 / 无激励 / 难认证"三个断点（故事钩子）。
2. **做什么** —— 用学分银行 + 哈希存证凭证 + Agent 把三个断点接起来（产品与创新）。
3. **怎么做** —— 清晰的技术选型与架构，证明它是个完整、可落地的工程（技术路线）。

**P27 · 深讲③ 区块链证书存证（SHA-256 哈希 + 扫码验真）**
- 说什么（全部有代码支撑，诚实口径）：
  - **存证怎么生成**：证书签发时（`LearningService.createCertificate`），用 **SHA-256** 对关键元数据 `certNo | userId | courseId | title` 做**单向哈希**，得到 64 位十六进制摘要，写入 `LearningCertificate.blockchainHash` 并随证书落库（`sha256()` 基于 `MessageDigest.getInstance("SHA-256")` + `HexFormat`，真实实现）。
  - **为什么防伪**：哈希**不可逆、且对原文极小改动极度敏感**——用同样算法对证书信息重算，摘要不一致即被篡改。
  - **扫码验真**：`qrContent` 存的是公开验真 URL（`/api/learning/certificates/verify?certNo=…&hash=…`），二维码内容即该 URL，扫码直达验真接口。
  - **验真逻辑**：`verifyCertificate(certNo, hash)` 按编号查证书，`valid = verifyStatus==1 且 传入 hash 与库内 blockchainHash 一致`；通过返回"区块链存证校验通过"，否则提示"证书编号或链上哈希不匹配"。
  - **存证链路**：`LearningCertificate` 与 `LearningAchievement` 同步冗余存同一哈希，形成"学习成果 → 证书 → 哈希摘要"的存证记录。
  - **诚实边界（必讲）**：这是 **SHA-256 链下哈希存证**，**未真正上链**——没有联盟链广播、没有智能合约、各证书哈希相互独立（未做哈希链式串联 / Merkle / 共识）。`verify` 接口已就绪，可平滑对接未来联盟链查询；本次实现的是"可验证防伪"这一环。
