# 个人总结与自我鉴定

这次大二实训，我主要参与的是「星秩存册」终身学习学分银行平台的开发与联调。项目本身模块不少，从前端页面到后端接口、数据库迁移，再到后来的云端部署，几乎都摸了一遍。刚开始几天其实有点懵，仓库分支又多，本地 Docker、端口、Redis 偶尔还会打架，经常是上午还在看别人合进来的代码，下午就得自己上手改导航、修路由。好在项目骨架比较完整，顺着学员、企业、管理员三条线走几遍，业务大概就清楚了：学习拿学分、账户流水、商城与求职、机构之间的学分互认，这些才是平台真正要讲清楚的故事。

实训中段我花了不少时间在体验和稳定性上。主题统一、对比度、活动状态流转、招聘投递到视频面试这一串，看起来不像“大功能”，但演示时特别容易露馅。我也接过 TRTC 和 AI 相关的配置，第一次把第三方密钥真正接到业务里，才明白文档写得再漂亮，还是得自己跑通才算数。印象最深的还是跨机构学分转换这一块。一开始规则里带着源类型、源标签去匹配，用着用着就觉得别扭——机构明明只能管自己的课，外部课程能不能认，本来就该人审，顶多让 AI 帮忙初筛。后来我们把流程改成机构只配目标课和规则说明，学员先选机构再选课程，提交时自动出初筛建议，企业端只看建议再点通过或驳回。改的时候踩过不少坑，比如表格操作列浮在上面挡住内容、按钮点完一直高亮、说明书导出成乱码，都是一点点对着现象查出来的。到最后把代码、演示数据和本地配置推到 Gitee，并整理服务器上的拉取覆盖步骤，才觉得这趟实训算是有头有尾。

回头看，最大的收获不是学会了某个具体接口怎么写，而是习惯了把需求先问清楚再动手，做完还要自己当用户走一遍。以前写作业往往停在“能跑”，现在更在意合不合业务、合不合演示。不足也很明显：有时候为了赶进度，注释和文档会滞后；对生产环境的安全和密钥管理还不够敏感，演示方便就把配置一并提交了，这在正式项目里是要检讨的。协作上，分支一多就容易乱，我也还在学怎么更干净地合并和回归。

总的来说，这十来天我把自己放进了一个相对真实的项目节奏里，有压力，也有做出东西的踏实感。如果后面还有机会继续维护这个平台，我希望能把互认规则和审核策略再打磨细一点，也把部署和文档写得更省事，让后来的同学少走一点弯路。
