## 资讯中心查询方式

资讯中心前端**分 Tab 展示**，后端分别查不同表：

```
招聘 Tab  → SELECT * FROM job_posting WHERE status=1
活动 Tab  → SELECT * FROM activity WHERE status IN (1,2)
政策 Tab  → SELECT * FROM policy_news WHERE status=1
```

## 标签对接（课程 ↔ 招聘 ↔ AI）

```
sys_tag ←→ course_tag ←→ course
sys_tag ←→ job_tag    ←→ job_posting
sys_tag ←→ user_target_tag ←→ user（心仪职位技能）
```

AI 推荐逻辑：比对 `user_target_tag` 与 `job_tag` / `course_tag` 计算契合度。

## 变更规范

1. 只在本 `db` 分支修改 SQL
2. 新增表/字段写入 `docker/mysql/migrations/00x_描述.sql`
3. 同步更新本文档
4. 合并到 `master` 后组员 `git pull` 并重建/迁移本地数据库
