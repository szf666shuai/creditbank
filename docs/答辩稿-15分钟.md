### 开场（约 40s）

各位老师好，我们是九组，答辩项目是**星秩存册——终身学习学分银行平台**。  
一句话定位：**它既是学习的“图书馆”，也是学习的“健身房”**——用可信、可累积、可跨机构流通的学分，把零散学习连成闭环。  
接下来约 8 分钟讲清「为什么做、做什么、怎么做」，再用约 6 分钟演示我负责的主界面、搜索与 Agent 相关核心能力。

### 为什么做（约 1.5 分钟）

终身学习有三个断点：  
**脱节**——课堂更新跟不上产业；**无氛围**——自学难坚持；**难认证**——学分散落各平台、企业不认。  
界面上我们用绿（成长）、蓝（信任）、奶油底（亲和）+ 新粗野风，对应“想留住学习者、又像学分银行一样可信”。  
价值飞轮是：学员来学 → 平台沉淀可信学分 → 企业招到人 → 更多机构入驻。

（背景/调研页可各 20 秒带过；功能矩阵强调「学习—档案—互认—求职」闭环即可。）

### 产品做什么（约 2 分钟）

三端：学员 / 企业机构 / 管理员。  
亮点我重点提和我相关的：  
1. **Agent 智能学伴**：对话答疑、学习画像、AI 简历、搜索侧智能解读，以及**学分互认 AI 初筛**——把大模型嵌进业务，而不是挂个聊天窗。  
2.（可一句）证书哈希存证、TRTC 视频面试——组内其他同学深讲/演示。  
完整度上：从学完拿证、攒秩点，到跨机构学分转换、再到求职，链路是通的。

### 怎么做——技术选型（你主讲，约 2.5 分钟）★

我们是**前后端分离的分层单体**（不是微服务）：  

| 层 | 选型 | 为什么选（口述） |
|----|------|------------------|
| 后端 | Java 21 + Spring Boot 3.4 + MyBatis-Plus | 生态成熟、开发效率高；MyBatis-Plus 分页/逻辑删除省样板代码 |
| 前端 | Vue 3 + TypeScript + Vite + Element Plus + Pinia | 组合式 API 适合复杂交互；Vite 热更新快；TS 降接口联调成本 |
| 数据 | MySQL 8 + Redis | 学分/档案等强一致业务用 MySQL；首页聚合用 Redis 缓存（约 5 分钟 TTL） |
| 部署 | Docker Compose（MySQL/Redis/Nginx） | 环境一致、答辩和云服务器一键起依赖 |
| 外接 | DeepSeek（OpenAI 兼容 Chat API）、腾讯云 TRTC | LLM 成本与中文效果合适；音视频不自研 SFU |

环境搭建路径：本机 `docker compose up` → `mvnw spring-boot:run` → `npm run dev`；云服务器可用 `scripts/deploy.sh`。配置在 `application.yml` + `application-local.yml`（LLM/TRTC 密钥）。

### A. 环境搭建与技术选型

**Q1：为什么前后端分离？为什么不是微服务？**  
A：三端（学员/企业/管理）共用一套 API，分离后前端用角色路由切换视图即可。体量上分层单体够用，Docker 起 MySQL/Redis/Nginx 就能演示；微服务会引入注册中心、拆分部署成本，现阶段收益不大。架构页我们明确说是**分层单体，可演进微服务**。

**Q2：为什么选 Vue3 + Spring Boot？**  
A：前端要强交互（搜索联想、Agent 面板、图表），Vue3 + Vite 开发体验好；后端事务、鉴权、对接第三方 HTTP（DeepSeek/TRTC）Spring Boot 成熟。TypeScript 保证接口字段和后端 VO 对齐。

**Q3：Redis 干什么用？挂了会怎样？**  
A：首页聚合接口 `HomeService.getHomeData()` 用 Redis 缓存（key 类似 `home:data:v1`，TTL 约 5 分钟），减轻多表查询。缓存失效或 Redis 短暂不可用时应回源数据库构建（实现上有缓存层，答辩可说“降级查库”）。  
路径：`backend/.../service/HomeService.java`，`controller/HomeController.java`。

**Q4：本地怎么起环境？**  
A：`docker/docker-compose.yml` 起 MySQL(3307)、Redis(6379)、Nginx(80) → `backend` 用 `mvnw spring-boot:run` → `frontend` 用 `npm run dev`。密钥在 `application-local.yml`（llm/trtc）。云服务器用 `scripts/deploy.sh`（云上前端 HTTPS，本地仍 HTTP）。

**选型金句（背一句）：**  
「选型服务于学分银行业务：强一致走 MySQL，读多聚合走 Redis，智能能力外接 LLM，音视频外接 TRTC，工程上 Docker 保证环境可复现。」

### B. 主界面开发

**Q5：主界面怎么按角色区分？**  
A：`HomeView.vue` 根据 `authStore` 的角色动态挂载 `StudentHomeView` / `EnterpriseHomeView` / `AdminHomeView`。布局是 `MainLayout` + `AppHeader` + `AppFooter`，Agent 宠物挂在布局里全局可用。

**关键路径：**  
- `frontend/src/views/HomeView.vue`  
- `frontend/src/views/home/StudentHomeView.vue`  
- `frontend/src/components/home/HomeContentSections.vue`  
- `frontend/src/components/layout/AppHeader.vue`  
- `frontend/src/layouts/MainLayout.vue`  
- 数据：`GET /api/home` → `HomeController` / `HomeService`

**Q6：首页内容从哪来？**  
A：后端 `HomeMapper` 聚合推荐课程、活动、资讯、招聘、论坛、机构等，封装成 VO；前端分区渲染，失败时可用 `home-fallback` 兜底，避免演示白板。

### C. 搜索相关

**Q7：搜索支持哪些类型？怎么实现的？**  
A：后端 `SearchService.search()` / `suggest()`，经 `SearchController` 暴露：  
- `GET /api/search?q=&type=&limit=`  
- `GET /api/search/suggest`  
`SearchMapper` 对课程、资源、论坛、资讯、活动、岗位、企业、学分等相关表做 LIKE 查询；suggest 对标题去重，限制条数（约 8 条）。  
前端顶栏 `useSearchSuggest.ts` **280ms 防抖**，结果页 `SearchView.vue`，分类配置在 `search-categories.ts`。

**关键路径：**  
- `backend/.../controller/SearchController.java`  
- `backend/.../service/SearchService.java`  
- `backend/.../mapper/SearchMapper.java`  
- `frontend/src/composables/useSearchSuggest.ts`  
- `frontend/src/views/SearchView.vue`  
- `frontend/src/api/search.ts`

**Q8：搜索和 Agent 怎么结合？**  
A：`SearchAgentPanel.vue` 把当前结果摘要作为 context，调同一套 `POST /api/agent/chat`，用户可要求“总结结果 / 学习路径建议”，避免搜索页只是冷冰冰的列表。

### D. Agent 全功能

#### D1. 统一 LLM 调用

**Q9：Agent 底层怎么调 DeepSeek？**  
A：`LlmService.chat()` 用 HTTP 调 `{baseUrl}/chat/completions`（OpenAI 兼容），配置来自 `LlmProperties` / `application-local.yml` 的 `llm.*`（provider、api-key、model=`deepseek-chat`）。没有单独引入官方 SDK，便于换兼容网关。  
路径：`backend/.../service/LlmService.java`，`config/LlmProperties.java`。

**Q10：对话如何“懂我的学习情况”？**  
A：`AgentService.chat()` 在用户已登录时调用 `LearningProfileService.buildChatContext(userId)`，把学分、进度、档案等拼进 context，再交给 `LlmService`。前端全局入口是 `AiAssistantPet.vue`（`MainLayout` 挂载），API 为 `frontend/src/api/agent.ts` → `/api/agent/chat`。  
路径：`AgentController` / `AgentService` / `AiAssistantPet.vue`。

#### D2. 学习画像

**Q11：学习画像怎么生成？**  
A：`LearningProfileService.generate()`：先 `loadSituation()` 聚合课程进度、档案、成就等事实 → 构造 prompt 让 LLM 输出 JSON 画像 → 落库 `UserLearningProfile`。页面在 `LearningProfileView.vue`，接口 `LearningProfileController`。  
要点：**先有结构化事实，再让模型总结**，减少胡编。

#### D3. AI 简历

**Q12：AI 简历和聊天是不是两套模型？**  
A：同一套 `LlmService`。`ProfileResumeService.generateResumeWithAi()` 根据档案/成就/用户信息拼 prompt，解析 JSON 后与用户资料 merge，接口 `POST /api/profile/resumes/ai-generate`。前端 `ResumeEditorView.vue` + `generateResumeAiApi()`。

#### D4. 学分互认 AI 初筛

**Q13：初筛是提交时自动的还是企业点按钮？**  
A：**学员 `apply` 成功后自动执行** `runAiScreenAndPersist()`；企业端主要展示 `aiSuggestion` / `aiReason` 做人审。初筛异常被吞掉，不影响申请落库。LLM 不可用时走 `heuristicScreen()` 启发式。  
路径：`CreditTransferService.apply()` → `runAiScreenAndPersist()` → `evaluateAiScreen()`；库表字段见迁移 `027_credit_transfer_ai_screen.sql`；前端学员 `CreditTransferView.vue`，企业 `TransferApplicationsView.vue`。

**Q14：AI 会不会直接通过学分？**  
A：不会。AI 只给 `approve/reject/uncertain` 建议和理由，**最终权限在企业审核**。这是人机协同，也符合学分银行“机构承认”的业务本质。

**Agent 金句（背一句）：**  
「我们把 LLM 接到四条业务链上：陪学对话、画像、简历、互认初筛——统一走 LlmService，上下文来自真实学习数据，而不是通用聊天机器人。」
