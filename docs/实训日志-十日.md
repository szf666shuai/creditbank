# 星秩存册 · 大二实训日志（12 天）

**实训项目**：星秩存册（Singularis Ledger）终身学习学分银行平台  
**技术栈**：Vue 3 + Spring Boot 3 + MySQL 8 + Redis + Docker  
**记录区间**：2026年7月7日 — 2026年7月18日  

---

## 第 1 天（2026-07-07）星期一

**今日任务**  
熟悉「星秩存册」项目整体架构与仓库结构，梳理学员 / 企业 / 管理员三端职责；阅读现有数据库初始化脚本与 README，完成本机 Docker（MySQL、Redis）与前后端启动环境核对。

**完成情况**  
1. 理清 `backend`、`frontend`、`docker` 目录职责与启动方式。  
2. 确认测试账号（学员、企业、管理员）与本地端口约定（前端 5173、后端 8080、MySQL 3307）。  
3. 初步列出待完善模块：搜索、个人中心、学分互认、招聘面试等。

**问题与解决**  
Docker 首次拉镜像较慢，通过镜像加速与耐心等待解决；确认 MySQL 映射为 3307 而非文档旧写的 3306，避免连错端口。

**收获**  
建立了对学分银行业务闭环的整体认识：学习获学分 → 账户/流水 → 商城/求职 → 机构互认。

## 第 10 天（2026-07-16）星期三

**今日任务**  
完成学分转换体验打磨、AI 初筛自动流转、文档与部署交付。

**完成情况**  
1. **导航**：调整顶栏疏密，缓解右侧拥挤。  
2. **企业规则**：仅保留目标课程、比例、规则说明；同一机构同一课程仅一条启用规则。  
3. **学员申请**：先选转入机构，再选目标课程，自动展示匹配规则（突出规则说明）。  
4. **AI 初筛**：学员提交时自动执行并落库；企业端展示建议标签与理由，去掉手动「AI 初筛」按钮；企业门户增加「待审核转换」快捷入口。  
5. **文档**：按真实库表重写《数据库设计说明书》，修复编码问题。  
6. **配置与发布**：配置 DeepSeek 与腾讯云 TRTC；将代码、本地配置与演示迁移 SQL 强制推送至 Gitee；整理云服务器拉取与运行步骤。

**问题与解决**  
1. 表格操作列 `fixed="right"` 遮挡内容 → 取消固定列。  
2. 链接按钮点击后高亮常亮 → 修正 `:focus` 样式并在点击后失焦。  
3. Word 说明书乱码 → 避免 PowerShell 管道转码，改用 UTF-8 直连导出重建。  
4. Gitee 远程跟踪分支不存在 → 先 `fetch` 再 `reset --hard`。

**收获**  
完成了从需求澄清 → 实现 → 联调 → 文档 → 远程部署指引的完整闭环；对「AI 辅助审核、人机协同」有了可演示的落地实践。

**完成情况**  
1. **学习档案与成果**：在「档案与成果」内增加「学习总览」分界面（折线图、柱形图、雷达图），进入该页默认先展示总览，再切到档案明细表；命名刻意避开与侧栏「学习画像」重名。  
2. **企业端权限**：企业管理员不可进入「企业中心」加盟列表；「企业运营」下拉去掉「加盟企业」；企业登录查看企业详情时隐藏「课程概览」Tab，避免出现「仅学员或管理员可浏览学习资源」报错。  
3. **文档与交付**：撰写实习（实训）体会；梳理提交源码应包含 / 应删除内容（如 `node_modules`、`target`）；确认 `application-local.yml`（DeepSeek / TRTC）已随仓库推送，云端拉取后无需再手填密钥。  
4. **环境**：本机重新拉起 Docker、前后端；了解公网 HTTP 下摄像头 / 麦克风受限的原因，以及域名 + HTTPS 的解决思路。  
5. **发布**：相关改动推送 GitHub `dev/jyw` 与 Gitee `master`，便于云服务器同步。

**问题与解决**  
1. 图表页若命名为「学习画像」会与 AI 画像功能冲突 → 改为「学习总览 / 学习数据总览」。  
2. 误以为 `docker` 目录丢失 → 从远程检出核对，实际与仓库一致。  
3. 提交包体积过大 → 明确删除前端 `node_modules`、后端 `target` 等依赖与编译产物。

**收获**  
答辩前更注重「演示路径是否干净」：角色权限、首屏信息、文档与仓库是否一致，比再堆新功能更重要；也进一步理解了浏览器安全上下文对音视频面试演示的影响。

**今日任务**  
项目展示与答辩：按学员 / 企业 / 管理员三条演示链路讲解「星秩存册」核心能力，回答老师关于技术选型、业务闭环与个人分工的问题。

**完成情况**  
1. **演示准备**：核对测试账号（`student1` / `enterprise1` / `admin`，密码 `admin123`），预跑学习档案总览、学分转换申请与 AI 初筛、企业审核、招聘面试与 TRTC、管理端审核等关键路径。  
2. **现场展示**：介绍平台定位（终身学习学分银行）与三端分工；重点演示学习获学分 → 档案可视化 → 跨机构学分互认（规则配置 / 学员申请 / AI 初筛 / 企业审核）→ 求职面试等闭环。  
3. **答辩说明**：说明技术栈（Vue 3 + Spring Boot + MySQL + Redis + Docker）、个人主要工作（导航与体验、学分转换与 AI 初筛、档案图表、企业权限收口、文档与 Gitee 部署交付）、遇到的问题与解决方式。  
4. **材料核对**：确认源码包已去掉过大依赖目录，说明书、实训日志、实习体会等文档齐备，仓库与本地演示版本一致。

**问题与解决**  
1. 演示环境若依赖本机 Docker，提前确认 MySQL / Redis 健康与前后端端口，避免答辩当场启动失败。  
2. 若使用云服务器公网 HTTP 演示视频面试，摄像头可能不可用 → 优先本机 `localhost` 演示，或提前说明需 HTTPS；必要时改用已准备好的录屏 / 备用链路。  
3. 老师追问细节时，按「业务问题 → 方案取舍 → 实现结果」回答，避免只堆技术名词。

**收获**  
把十几天的开发收束成一条可讲清的故事：学分如何产生、如何看见、如何跨机构互认、如何服务求职。实训不只是把功能做出来，更要能讲清楚「为什么这样设计、怎么验证、还有什么不足」。
