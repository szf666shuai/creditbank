# 星秩存册 · 大二实训日志（12 天）

实训项目：星秩存册（Singularis Ledger）终身学习学分银行平台  
技术栈：Vue 3 + Spring Boot 3 + MySQL 8 + Redis + Docker  
记录区间：2026年7月7日 — 2026年7月18日

---

## 第 1 天（2026-07-07）星期一

今天主要是把项目摸清楚。仓库里 backend、frontend、docker 各管一块，我对着 README 和初始化脚本，把学员、企业、管理员三条线的职责梳了一遍，也把本机 Docker 里的 MySQL、Redis 和前后端启动方式核对了一下。第一次拉镜像有点慢，加了镜像源之后才顺；另外发现文档里还有人写着 3306，实际映射是 3307，差点连错库。晚上大致有了数：学习拿学分、账户流水、商城求职、机构互认，大概就是这条闭环。测试账号和端口也记下来了——前端 5173、后端 8080、库 3307。

---

## 第 2 天（2026-07-08）星期二

开始动首页和导航。轮播、内容分区、顶栏交互一点点接上，全局搜索和分类 Tab 也做了，结果页可读性顺手改了一下。种子数据从分支同步进 init，保证演示时搜得到东西。分类 Tab 顺序一开始和产品预期对不上，调过之后体验才统一。做完才觉得，首页入口再好看，没有好用的搜索，平台还是不好使。

---

## 第 3 天（2026-07-09）星期三

搜索继续打磨，加了联想补全，结果页也再收拾了一遍。把 mainform 主界面和搜索合回主线时冲突不少，只能按文件对比业务意图，能留的本地商城和导航改动都留住。学分规则相关种子也补上了，后面演示积分规则才不至于空。今天对 Git 协作的感受比写功能还深——分支一多，合并本身就是工作量。

---

## 第 4 天（2026-07-10）星期四

三端个人中心和角色权限修了一轮，同时把 DeepSeek 助手、搜索对话和基于登录用户的学习画像接进来。学习成果缺 orgId 时后端直接空指针，补了判空才稳住；模型返回结构偶尔飘，前端也加了容错，免得演示当场白屏。JWT 拦角色这件事以前知道概念，这次才真正碰到“不拦就会串数据”。大模型进业务也不是贴个聊天框那么简单，上下文和展示都要自己兜。

---

## 第 5 天（2026-07-11）星期五

品牌和商城、秩点相关改动推进了一截，论坛和资讯也合进来了，布局按页面做了区分，学习统计、活动深链一并补上。合并最麻烦的是路由和商城入口被盖掉，只能针对性恢复再整条回归。今天最大的教训是：大合并之后如果不冒烟走一遍，后面返工更狠。视觉统一之后，答辩观感明显好一截。

---

## 第 6 天（2026-07-12）星期六

没有猛堆新功能，而是按学员、企业、管理员把登录、学习、商城、论坛、资讯、个人中心走查了一遍，把对比度不够、入口不明显、状态不同步记成清单，方便下周改。Redis 一度被本机另一个无密码实例占着，后端连不上，关掉冲突进程才恢复。慢慢习惯了：功能做完先当用户点一遍，比闷头往下写省事。

---

## 第 7 天（2026-07-13）星期日

活动状态流转、评论点赞和秩点相关接口修了几处，招聘投递状态也往完整闭环靠。TRTC 视频面试接进来之后，前后端对房间凭证和时间窗口的约定必须对齐，配置统一后才不踩坑。管理端审核、机构注册和界面又改了一轮，并同步了 master。主题和业务页风格还不统一，只能继续往一套视觉上收。第一次把 AppId、UserSig 真正跑通，对“音视频不是前端单独的事”有了实感。

---

## 第 8 天（2026-07-14）星期一

浅色 Neo-Brutal 主题基本铺开，低对比文字修了一批，顺手清了不少体验类小问题，业务闭环看起来像样多了。学分银行概念模型和物理模型的说明也写了，后面答辩讲结构不至于只靠口头。论坛分支又合了一次。全局样式碰到 Element Plus 的焦点态有点别扭，链接按钮点完还亮着的问题后面继续查。有文档衬着代码，讲起来确实踏实一些。

---

## 第 9 天（2026-07-15）星期二

开始啃跨机构学分转换。先把学分、活动、诚信和个人中心的数据流理清，再定方案：机构配接收规则，学员发起转入，企业审核。首页和个人中心怎么露出“学分银行”能力也想了一版，演示数据和学员、企业联调用例一并准备。早期规则靠源标签匹配，用着用着就别扭——机构其实管不了外面的课，外面认不认该人审或 AI 帮着看。后来改成机构只配目标课和说明，这条抽象才清楚：规则是声明，申请是举证，审核才是承认。

---

## 第 10 天（2026-07-16）星期三

学分转换按昨天的方向收口。顶栏疏密调了一下，右边不那么挤；企业规则只留目标课程、比例和说明，同一机构同一课只保留一条启用规则。学员申请改成先选机构再选课，规则说明会自动带出来。提交时自动跑 AI 初筛并落库，企业端看建议和理由就行，手动初筛按钮拿掉了，门户上加了待审核入口。说明书按真实库表重写，编码问题折腾了一阵——PowerShell 管道容易把中文搞乱，后来 UTF-8 直连导出才正常。DeepSeek 和 TRTC 配置进仓库，代码和演示 SQL 推到 Gitee，服务器拉取步骤也整理好了。表格操作列挡住内容、按钮点完一直高亮，都是演示时露馅的小坑，对着现象一点点修。这一天算是把“说清楚、做出来、能部署”串起来了。

---

## 第 11 天（2026-07-17）星期五

答辩前再收一轮。档案与成果里加了学习总览，折线、柱形、雷达图先看整体，再切明细表；名字故意不用“学习画像”，免得和侧栏 AI 画像撞车。企业管理员进不了加盟企业列表，运营下拉里的加盟入口也去掉，看自己企业详情时不再露课程概览，省得权限报错扫兴。体会写了，提交源码该留该删也列清楚，node_modules 和 target 这种千万别打进包里。确认 local 配置已经在仓库里，云上拉取一般不用再手填密钥。本机环境重新拉起过，也搞明白了公网 HTTP 下摄像头麦克风为什么受限。改动推到 GitHub 和 Gitee，方便服务器同步。有一阵子以为 docker 目录丢了，对远程一看还在，虚惊一场。越到后期越觉得，演示路径干净比再堆功能重要。

---

## 第 12 天（2026-07-18）星期六

今天是项目展示和答辩。提前把 student1、enterprise1、admin 路径预跑了一遍，档案总览、学分转换和 AI 初筛、企业审核、面试和管理端该亮的都确认能亮。现场按学分银行的故事讲：学分怎么来、怎么看见、怎么跨机构互认、怎么接到求职。技术栈和自己负责的主界面、搜索、Agent、互认初筛、文档部署，按老师问题回答。材料上源码包去掉了过大的依赖目录，说明书和日志体会也齐。环境上提前看了 Docker 和端口，视频面试优先本机演示，避免公网 HTTP 卡住摄像头。答辩让我更清楚一件事：功能做出来只是一半，能讲清为什么这样设计、怎么验证、还有什么不足，才算把这趟实训收住。

---

## 这十二天整体感受

业务上，学习、商城、论坛资讯、招聘面试和跨机构学分互认这条演示链路基本打通，档案总览和企业端权限也收干净了。技术上从前端 Vue3、后端 Spring Boot，到 MySQL 迁移、Redis、TRTC、大模型初筛和图表，都真正碰过。工程上分支合并、Docker、Gitee 发布和云服务器拉取反复练过；文档和答辩材料也跟着补齐。后面若继续维护，密钥和 HTTPS、初筛异步队列、诚信策略和常驻部署脚本都还可以再打磨。
