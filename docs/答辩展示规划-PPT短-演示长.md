## 三、项目演示主线（10～12 分钟）——推荐版

用一条故事串起来，避免东点西点：

> **学员进来 → 看见平台 → 搜到课 → 问懂 AI → 生成画像/简历 → 申请学分互认（AI 初筛）→ 企业看见建议**

### 分段节奏（合计约 11 分钟）

| 段 | 内容 | 时长 | 账号 | 操作要点 | 边点边说（一句） |
|----|------|------|------|----------|------------------|
| **B0** | 开场 | 20s | — | 浏览器已登录或马上登 | 「演示按学分银行闭环：看得见、搜得到、问得懂、转得动。」 |
| **B1** | 主界面 / 首页 | **70s** | `student1` | 首页逛一圈：导航、分区、学分氛围 | 「三端合一 SPA；首页 `/api/home` 聚合，Redis 缓存。」 |
| **B2** | 全局搜索 + 联想 | **90s** | 同上 | 顶栏输入「Java」→ 联想 → 回车进结果页 → 切 1～2 个分类 Tab | 「多类型检索；联想防抖；结果页可继续用 AI。」 |
| **B3** | 搜索侧 Agent | **70s** | 同上 | 右侧点「帮我总结这些结果」或「学习路径建议」 | 「同一套 `/api/agent/chat`，上下文是当前搜索摘要。」 |
| **B4** | 全局 Agent 对话 | **90s** | 同上 | 双击右下角宠物 → 问：「根据我的学习情况，下一步学什么？」 | 「后端会注入学习档案上下文，再调 DeepSeek。」 |
| **B5** | 学习画像 | **80s** | 同上 | 个人中心 → 学习画像 → 生成/刷新 → 指摘要、技能、建议 | 「先聚合库里的事实，再让模型输出结构化 JSON。」 |
| **B6** | 档案总览（可选加分） | **50s** | 同上 | 档案与成果 → **学习总览**图表 → 再切明细 | 「学习情况可视化；和侧栏 AI 画像不是同一个功能。」 |
| **B7** | AI 简历 | **80s** | 同上 | 我的简历 → AI 生成 → 展示草稿可编辑 | 「LLM 能力复用到简历字段，再和真实资料合并。」 |
| **B8** | 学分互认 + AI 初筛 | **2.5～3 分钟** | 先学员后企业 | 见下方细拆 | 「提交自动初筛；企业人审拍板。」 |
| **B9** | 收束 | 20s | — | 回到首页或停在企业审核页 | 「以上是我负责的主界面、搜索与 Agent 链路。请老师提问。」 |

**B1～B9 合计约 11 分钟**（B6 可砍，则约 10 分钟）。

### B8 学分互认细拆（最出彩，务必练熟）

**学员侧（约 90s）——`student1`**

1. 个人中心 → 学分转换（或你菜单里的入口）  
2. 选转入机构 → 选目标课程 → 看规则说明自动出来  
3. 填转入理由 → 提交  
4. 口述：「申请已落库；后端 `apply` 里同步调 AI 初筛。」

**企业侧（约 70～90s）——退出换 `enterprise1`**

1. 企业工作台 / 转换申请  
2. 打开刚提交或已有待审记录  
3. **指给老师看：AI 建议标签 + 理由**  
4. 口述：「建议是 approve/reject/uncertain；我点通过或驳回才算数，AI 不替企业做决定。」

**省时间技巧：**  
若现场 DeepSeek 慢，提前用 `student1` 交好一条申请，演示时学员侧只「走一遍表单」或直接切企业端讲已有 AI 结果；并说明「初筛在提交时自动跑，LLM 失败有启发式兜底」。

## 六、演示串词（可直接念，约 11 分钟语速）

**【B0】** 老师们好，下面进入系统演示。我们按「看得见 → 搜得到 → 问得懂 → 转得动」来走。

**【B1】** 学员登录后是主界面。顶栏导航和搜索，中间是学分银行相关内容。这是 Vue 三端合一的学员首页，数据来自后端首页聚合接口，带 Redis 缓存。

**【B2】** 顶栏搜「Java」，先出联想，回车进结果页，课程、资讯等可以分类看——这是全局搜索。

**【B3】** 右侧助手可以基于**当前这些结果**做总结或路径建议，上下文是搜索摘要，接口仍是统一的 Agent 对话。

**【B4】** 双击右下角学伴，我问一句和个人相关的。后端会把我的学分、课程进度等拼进上下文，再调 DeepSeek。

**【B5】** 学习画像：先拉库里的学习事实，再生成结构化摘要和建议，并落库，之后对话也能用到。

**【B6 可选】** 档案与成果里的学习总览，用图表看学习情况，和 AI 画像是分开的功能。

**【B7】** AI 简历：同一套大模型能力，按档案生成可编辑草稿。

**【B8】** 学分转换：学员选机构、目标课、提交；企业端能看到 AI 初筛建议和理由，最终由企业审核。这体现学分银行「机构承认」+ 人机协同。

**【B9】** 以上是主界面、搜索与 Agent 相关核心能力。我的讲解到这里，请老师批评指正。
