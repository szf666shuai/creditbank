# 实习（实训）体会

这次大二实训，我参与的是「星秩存册」终身学习学分银行平台的开发。项目不大不小，却把学员、企业、管理员三条线都串在一起，从本地把 Docker、前后端跑通，到后来改导航、修路由、做学分转换，再到推到 Gitee、整理服务器拉取步骤，十来天里节奏一直比较紧。刚进组那几天其实挺慌的，仓库分支多，文档和实际端口偶尔也对不上，上午还在摸别人合进来的代码，下午就得自己动手改页面。慢慢顺着业务走几遍才明白：平台要讲清楚的，不只是“能登录能上课”，而是学习拿学分、账户流水、商城求职、机构之间学分互认这一整条故事。

说到对企业文化的感受，实训里接触的不完全是传统意义上的“公司文化”，更像是一个真实项目组在怎么做事。大家默认要把演示跑通、把角色权限分清楚，遇到冲突也不是谁嗓门大谁说了算，而是对着业务意图一点点对。我印象比较深的是改学分转换规则那段时间：一开始规则里带着源类型、源标签去匹配，用着用着就觉得别扭——机构明明只能管自己的课，外面的课能不能认，本来就该人来审。后来我们反复讨论，把流程收成机构只配目标课和规则说明，学员先选机构再选课程，提交时自动出 AI 初筛建议，企业端看着建议再点通过或驳回。这种“先问清楚再动手、做完还要当用户走一遍”的习惯，比单纯赶功能更像在正经做事。

专业技能方面，实训效果比我原来预想的实在得多。以前写作业，前端改改样式、后端调调接口就算交差；这次则是 Vue 3 和 Spring Boot 一起联调，还要碰 MySQL 迁移、Redis 端口冲突、TRTC 视频面试配置、DeepSeek 接入这类“文档好看、自己跑才算数”的东西。导航疏密、主题对比度、表格操作列挡住内容、按钮点完一直高亮，这些看起来都是小问题，演示时却特别容易露馅。我也第一次真正把第三方密钥接到业务里，才体会到联调不是把配置贴进去就行，前后端约定、空值防御、错误提示都得跟上。数据库说明书导出乱码那次，折腾了好一会儿，才知道 PowerShell 管道转码会把中文搞坏，后来改成 UTF-8 直连导出才稳住——这种坑书本里很少写，却很真实。

团队协作这块，我感触也挺深。分支一多就容易乱，合并时路由被覆盖、商城入口丢了都发生过，后来大家慢慢形成习惯：合完先按学员、企业、管理员三条路径冒烟走一遍，再往下开发。有些问题不是一个人能关起门来解决的，比如活动状态流转、招聘投递到视频面试、跨机构互认的产品口径，都要前后端和产品意图对齐。协作里我也看到自己的不足：有时候为了赶进度，注释和文档会滞后；演示方便就把本地配置一并提交，正式环境里这其实不够妥当。这些都提醒我，协作不只是“我改完你接着改”，还包括把环境、约定和回归清单交代清楚，让别人少踩坑。

创新创业技能方面，学分银行这个题目本身就带着一点“怎么把学习成果变成可流通价值”的味道。实训里最有意思的，是把 AI 从“聊天展示”推到“审核辅助”：学员提交转换申请时自动初筛落库，企业端看到建议标签和理由，人仍然握有最终决定权。这种人机协同的方式，让我意识到创新不一定是另起炉灶做一个很炫的功能，也可以是把已有流程里最费时、最容易扯皮的环节变轻一点。答辩和演示时，企业门户上的“待审核转换”入口一亮出来，业务闭环就清楚多了——创业也好、做产品也好，最后还是要回到“用户能不能顺畅走完一步”上。

职业素养上，这次锻炼最大的变化，是我开始在意“合不合业务、合不合演示”，而不只是“代码能不能跑”。走查时会主动找对比度不足、入口不明显、角色不该看的页面却能进这类问题；企业管理员不该进企业中心、不该看自己企业的课程概览，我们也按权限收干净了。部署交付那天，把代码、演示数据和服务器拉取步骤整理出来，才觉得这趟实训有头有尾。如果后面还有机会继续维护这个平台，我希望把互认规则和审核策略再打磨细一点，也把密钥管理和部署文档写得更省事，让后来的同学少走一点弯路。

总的来说，这十来天压力不小，但做出东西、把链路跑通的踏实感也很强。实训把我从“会写一点作业代码”往“能在真实项目节奏里协作交付”推了一截，这份体会我会带着继续往前走。
