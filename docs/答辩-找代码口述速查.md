# 答辩现场 · 功能实现速查（找代码口述）

用法：老师问「XX 怎么实现的」→ 按下表打开文件 → 指着关键方法讲下面「口述稿」。  
先打开后端再打开前端；一次打开 **2～3 个文件** 即可，别翻太多。

IDE 建议：Ctrl+Shift+F 全局搜表里的「搜索关键词」。

## 1. Agent / AI 相关（你主责，优先背熟）

### 1.1 「你们 AI / Agent 底层怎么调大模型的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `backend/src/main/java/com/creditbank/platform/service/LlmService.java` | 方法 `chat(...)` |
| ② | `backend/src/main/java/com/creditbank/platform/config/LlmProperties.java` | `prefix = "llm"` |
| ③ | `backend/src/main/resources/application-local.yml` | `llm.api-key` / `base-url` / `model` |

**搜索关键词：** `class LlmService` 或 `chat/completions`

**口述：**  
配置在 `llm` 段。`LlmService.chat` 把 system（可拼业务上下文）、最近最多 12 轮 history、当前 user 拼成 messages，用 Spring RestClient POST 到 DeepSeek 的 `/chat/completions`，再取 `choices[0].message.content`。没单独引官方 SDK，换兼容网关只改配置。

---

### 1.2 「右下角那个助手 / 对话怎么实现的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `frontend/src/layouts/MainLayout.vue` | 里有 `<AiAssistantPet />` |
| ② | `frontend/src/components/agent/AiAssistantPet.vue` | 发消息处 `agentChatApi` |
| ③ | `frontend/src/api/agent.ts` | `agentChatApi` → `POST /agent/chat` |
| ④ | `backend/.../controller/AgentController.java` | `@PostMapping("/chat")` |
| ⑤ | `backend/.../service/AgentService.java` | 方法 `chat`（注入学习上下文） |

**搜索关键词：** `AiAssistantPet` 或 `AgentService`

**口述：**  
布局全局挂宠物组件；用户发消息走 `agentChatApi`。后端 `AgentController` 进 `AgentService.chat`：校验长度后，若已登录就调 `LearningProfileService.buildChatContext` 拼个人学习情况，再调 `LlmService`。所以回答能贴合学分、课程进度，不是裸聊天。

### 1.3 「学习画像怎么生成的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `backend/.../controller/LearningProfileController.java` | `POST /generate` |
| ② | `backend/.../service/LearningProfileService.java` | `loadSituation` → `generate` |
| ③（可选） | `frontend/src/views/profile/LearningProfileView.vue` | 生成按钮 |
| ④（可选） | `frontend/src/api/learning-profile.ts` | `generateLearningProfileApi` |

**搜索关键词：** `LearningProfileService` 或 `GENERATE_PROMPT`

**口述：**  
`generate` 先 `loadSituation` 从库聚合课程进度、档案、成果、诚信分等事实，再带固定 JSON prompt 调 `LlmService`，解析后写入 `UserLearningProfile`。要求模型别编造上下文没有的课。对话时用的 `buildChatContext` 就是画像事实的精简版。

### 1.4 「AI 简历怎么实现的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `backend/.../module/profile/controller/ProfileResumeController.java` | `POST /ai-generate` |
| ② | `backend/.../module/profile/service/ProfileResumeService.java` | `generateResumeWithAi` |
| ③ | `frontend/src/api/profile-resume.ts` | `generateResumeAiApi` |
| ④（可选） | `frontend/src/views/profile/ResumeEditorView.vue` | AI 生成入口 |

**搜索关键词：** `generateResumeWithAi` 或 `ai-generate`

**口述：**  
`buildLearningContext` 拼用户信息和学习档案/成果，prompt 要求只输出简历 JSON 字段，`llmService.chat` 拿到结果后 `parseGeneratedContent`，再 `mergeWithUserProfile` 把真实手机邮箱盖回去，前端可继续编辑保存。和对话共用同一套 `LlmService`。

### 1.5 「搜索页旁边那个 AI 是什么？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `frontend/src/views/SearchView.vue` | 引入 `SearchAgentPanel` |
| ② | `frontend/src/components/agent/SearchAgentPanel.vue` | `buildContext` + `agentChatApi` |
| ③ | （后端仍是）`AgentService.java` | 同一 `/api/agent/chat` |

**搜索关键词：** `SearchAgentPanel` 或 `buildContext`

**口述：**  
没有单独后端服务。前端把当前关键词、分类、前几条搜索结果摘要拼成 `context`，仍调 `/api/agent/chat`；后端还会再叠学习画像上下文。快捷问题比如「总结这些结果」「给学习路径建议」。

---

### 1.6 「学分互认 AI 初筛怎么做的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `backend/.../service/CreditTransferService.java` | `apply` 末尾 `runAiScreenAndPersist` |
| ② | 同一文件 | `evaluateAiScreen` / `parseAiScreen` / `heuristicScreen` |
| ③（可选） | `docker/mysql/migrations/027_credit_transfer_ai_screen.sql` | AI 字段 |
| ④ 前端学员 | `frontend/src/views/profile/student/CreditTransferView.vue` | 提交申请 |
| ⑤ 前端企业 | `frontend/src/views/profile/enterprise/TransferApplicationsView.vue` | 展示建议标签 |

**搜索关键词：** `runAiScreenAndPersist` 或 `evaluateAiScreen`

**口述：**  
学员 `apply` 申请先落库，再 try-catch 调 AI 初筛，失败不影响申请。`evaluateAiScreen` 用机构规则说明 + 源课/目标/理由拼 prompt，让模型输出 approve/reject/uncertain 和理由，写入申请表。LLM 挂了走 `heuristicScreen`。企业端只看建议，**最终通过/驳回仍是人审**。

## 2. 搜索（非 AI 检索本身）

### 2.1 「全局搜索 / 联想怎么实现的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `frontend/src/components/layout/AppHeader.vue` | 搜索框 |
| ② | `frontend/src/composables/useSearchSuggest.ts` | 防抖联想 |
| ③ | `frontend/src/api/search.ts` | `searchApi` / `searchSuggestApi` |
| ④ | `backend/.../controller/SearchController.java` | `/search`、`/suggest` |
| ⑤ | `backend/.../service/SearchService.java` | `search` / `suggest` |
| ⑥（可选） | `backend/.../mapper/SearchMapper.java` | 各类型 LIKE SQL |

**搜索关键词：** `SearchService` 或 `useSearchSuggest`

**口述：**

### 3.1 「首页 / 主界面怎么做的？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|

## 4. 环境与技术选型（不一定打开很多代码）

### 4.1 「环境怎么搭的？技术栈为什么这样选？」

| 打开顺序 | 路径 | 指给老师看 |
|----------|------|------------|
| ① | `docker/docker-compose.yml` | mysql / redis / nginx |
| ② | `backend/pom.xml` | Spring Boot / MyBatis-Plus |
| ③ | `frontend/package.json` | Vue3 / Vite / Element Plus |

**口述：**

## 5. 老师追问时的「诚实三句」

## 6. 现场操作习惯（很重要）

1. **先复述问题**：「您问的是学习画像生成，对吧？」  
2. **先打开后端核心类**，指方法名，再打开前端调用点。  
3. **顺着调用链讲**：页面 → api/*.ts → Controller → Service →（AI 则）LlmService。  
4. 卡壳时全局搜方法名：`generateResumeWithAi`、`buildChatContext`、`runAiScreenAndPersist`。  
5. 不要一上来翻 node_modules 或大段 CSS。
