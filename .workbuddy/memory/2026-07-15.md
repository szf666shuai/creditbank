## PPTX 原生动画注入（用户要求「转成PPT后动画能否保留」）

**结论**: html2pptx 只处理 rotate/text-transform，**不解析 CSS 动画**，所以直接转会丢动画。但 PPTX 格式本身支持动画（进入/退出/强调 XML）。

**交付**: `deck-export/inject_animations.py` + 生成的 `星秩存册-界面设计答辩-动画版.pptx`（119 KB，28 页，共 482 个原生 fade 入场动画）。
- 技术：python-pptx 在每页 `<p:sld>` 注入 `<p:timing>`，每个前景形状一个 fade animEffect + `nodeType="withEffect"` 错峰延迟（140ms/个）依次出现；跳过全屏背景(无文字)避免整页闪白。
- 校验：zip 完整、python-pptx 可打开、28 页均含 timing、spTgt spid 全部有效、510 形状数与原 editable 版一致（内容未丢）。
- 局限：PPT 动画是一次性触发，无法还原 CSS 的 floatBob/pulseGlow/进度条填充等**循环动画**；真实截图只存在于 HTML 版，该 PPTX 未含截图（P16 为占位）。
